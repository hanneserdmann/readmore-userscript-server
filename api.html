Readmore-Userscript Server

  • Optionen synchronisieren

  • GET  :: /api/options/:id/:hash
    Liefert die gespeicherten Optionen als JSON String zurück. Falls etwas schiefgelaufen ist, wird ein entsprechendes Error-Object zurückgegeben.
  • POST :: /api/options/:id/:hash
    Erstellt einen neuen Eintrag für den User in der Datenbank und speichert die mitgeschickten Optionen. Falls bereits ein Eintrag vorhanden ist, wird ein entsprechendes Error-Object zurückgegeben.
  • PUT  :: /api/options/:id/:hash
    Aktualisiert einen bereits existierenden Eintrag für den User. Falls etwas schiefgelaufen ist, wird ein entsprechendes Error-Object zurückgegeben.
